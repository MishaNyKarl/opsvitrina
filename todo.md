  - Если пользователь пришел на статью с `?clickid=...&cost=...&campaign=...`, этот query string передается в ссылку клика по баннеру.
  - Backend redirect баннера добавляет query string к целевой ссылке.

- [x] Добавлять отдельную UTM-метку баннера ко второй ссылке.
  - У каждого баннера есть `utm_key`.
  - Сейчас она добавляется как `banner_utm`.
  - Также добавляется `banner_id`.

- [x] Довести проклейку до формата баера из примера.
  - Для обычных ссылок без `{...}` система сохраняет legacy-режим: добавляет весь хвост первой ссылки.
  - Для ссылок с `{...}` включается шаблонный режим: значения подставляются из первой ссылки без дублей всего хвоста.
  - Поддерживаются шаблоны во второй ссылке:

```text
https://lifestoryhub.net/click?key=...&click_id={click_id}&cost={cost}&ad_name={ad_name}&campaign={campaign}&state_id={state_id}&ad_vtr_name={ad_vtr_name}
```

  - `{click_id}`, `{cost}`, `{ad_name}`, `{campaign}`, `{state_id}` берутся из первой ссылки.
  - `{click_id}` также умеет брать значение из `phx_click_id`, `clickid` или `subid`.
  - `{ad_vtr_name}` подставляется из UTM/tracking key конкретного баннера.
  - Основное имя параметра баннера: `ad_vtr_name`.

## 8. Баерский сценарий ссылок

Текущий ожидаемый сценарий от баера:

1. Ссылка на вход в витрину:

```text
https://lifestoryhub.net/click?key=d6243ebecefe4fb0a204870793ab6fff&phx_click_id={12345}&cost={cost}&ad_name={12}&campaign={1111}
```

2. Первый клик со статьи должен дописать ID статьи:

```text
https://lifestoryhub.net/click?key=d6243ebecefe4fb0a204870793ab6fff&phx_click_id={12345}&cost={cost}&ad_name={12}&campaign={1111}&state_id={tz_top_5}
```

3. Ссылка на баннер выглядит так:

```text
https://lifestoryhub.net/click?key=c9788156bac94d389bf23e1a4bb603fa&click_id={click_id}&cost={cost}&ad_name={ad_name}&campaign={campaign}&state_id={state_id}&ad_vtr_name={ad_vtr_name}
```

4. В момент клика по баннеру значения из первой ссылки должны подставиться во вторую:

```text
ad_name={12}
campaign={1111}
state_id={tz_top_5}
```

5. Дополнительно должна подставиться метка баннера:

```text
ad_vtr_name={TZ_Prost_1}
```

6. Конечная ссылка должна стать:

```text
https://lifestoryhub.net/click?key=c9788156bac94d389bf23e1a4bb603fa&click_id={click_id}&cost={cost}&ad_name={12}&campaign={1111}&state_id={tz_top_5}&ad_vtr_name={TZ_Prost_1}
```

Что уже есть:

- [x] Хвост первой ссылки проклеивается ко второй.
- [x] У баннера есть отдельный tracking key.
- [x] Клик по баннеру идет через redirect endpoint.

Что уже сделано:

- [x] Поддержать шаблоны `{param}` во второй ссылке баннера.
- [x] Настроить имя параметра баннера: используется `ad_vtr_name`.
- [x] Подставлять `utm_key` баннера в `{ad_vtr_name}`.
- [ ] Уточнить соответствие `phx_click_id` из первой ссылки и `click_id` во второй ссылке.
